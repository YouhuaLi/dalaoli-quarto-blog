一个可以计算古德斯坦序列的2-色图灵机程序

每周一题
Published

October 13, 2020

简单版： 我写了一个可以计算古德斯坦序列的2-色图灵机程序。这个程序的初值是947，它恰有947个状态。所以它也同时证明了Rado的ΣΣ函数中，

Σ(947)>G(947)(947)>G(947)Σ(947)>G(947)

Σ(n)(n)Σ(n) 的定义为：所有可停机的2色-n状态图灵机中，停机后纸带上打印最多的“1”的那台图灵机，所打印的 “1”符号数量。 ΣΣ函数也被称为Busy Beaver function/忙碌海狸函数。

G(n)是古得斯坦序列数。

完整版：

2016年，MIT的博士生Adam Yeddida发布了一个通用的2-色图灵机编程语言: laconic。他同时发布了一个7910状态的2-色图灵机程序。这个程序可以证明ZFC的一致性，则同时根据哥德尔不完备定理，在ZFC公理系统中，无法证明这个图灵机会停机。

后来， github用户sorear发布了另一个通用图灵机编程语言NQL，用这种语言实现的图灵机会比laconic具有更少的状态数。他先后发布了一个1919状态和748状态的验证ZFC一致性的图灵机。

以上两位的程序设计目标都在于设计出状态数最少的，可证明ZFC一致性的图灵机。其潜在目标在于发现ZFC中可计算数的上限，比如按Sorea的结果，上界目前为Σ(748)(748)Σ(748)。

但我们也可以反过来思考。如果用以上语言执行一个已经证明会停机的算法，则可以用来估计Σ(n)(n)Σ(n)的下界。比如，如果一个图灵机程序可以计算一个很大的数x，这个图灵机有n状态，则证明Σ(n)>x(n)>xΣ(n)>x。

之前，对比较大的n，只知道一些很弱的下界：

Σ(2k)>3↑k−23>Ack(k−2,k−2) for k≥2(2 k)>3 ^{k-2} 3>Ack(k-2, k-2) k Σ(2k)>3↑k−23>Ack(k−2,k−2) for k≥2 其中Ack(m, n)是阿克曼函数。另外也知道Σ(17)>G64(17)>G_{64}Σ(17)>G64​，其中G64G_{64}G64​是葛立恒数。

我开始时尝试用laconic写了个计算阿克曼函数的算法，但结果表明阿克曼函数太弱了，而结果的状态数又太多，所以达不到改进Σ(n)(n)Σ(n)下界的目的。

后来，我发现古德斯坦序列是一个不错的选择。古德斯坦变换本身已经强到无法在皮亚诺算数公理中证明其有限。一些已知的古德斯坦序列下界非常大，比如： G(8)>Ack(Ack(3×2402,653,211))G(8)>Ack (Ack (3 ^{402,653,211}))G(8)>Ack(Ack(3×2402,653,211))

所以，我开始考虑用Laconic和NQL写计算古德斯坦序列的程序。虽然网上有现成的计算G(n) 的算法，但那些都用到了递归。而Laconic和NQL不支持递归函数，NQL甚至没有内置列表结果，所以我的编程也颇费功夫。

我先写了一个等价的python程序，验证算法，然后再移植入Laconic和NQL。在Laconic中，可以对初值4以内进行一次古德斯坦变换的计算：

time python2 laconic_interpreter.py goodstein s: 2 s: 3 s: 26 Program halted without errors.

real 0m9.745s user 0m8.845s sys 0m0.252s

其中的26，就是初值为4时，进行一次古德斯坦变换后的结果。 NQL中由于运行过慢，我没法等到哪怕初值为1的计算结果，如果有谁能帮我验证下，我不胜感激。

而在最终的NQL程序中，不同的初值，会导致最终状态数的少量变化。我发现恰好初值为947时，得到一个947状态的图灵机，因此最终提交了初值为947的代码：

proc main() { n = 947; base = 2; s = 1; while( s > 0 ){ stein_none_rescursive(n, base, s); incr(base); n = s; } return; } 未来的想法：
  • 简化程序，将程序状态缩小到748以内。那么这个程序就将是目前已知状态数最少的，ZFC无法证明其停机的图灵机程序。
  • 计算更大的数字。如果有某个大数，可以用更简单的算法算出更大的结果，那么它将很值得用NQL编程计算。如此可得到Σ(n)(n)Σ(n)函数更好的下界。
